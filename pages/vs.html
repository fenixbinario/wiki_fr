MVC vs MVVM

¿Qué es MVC?

MVC es un patrón arquitectónico que separa una aplicación en tres componentes lógicos principales: Modelo, Vista y Controlador. De ahí la abreviatura MVC. El MVC de forma completa es Model View Controller.
En esta arquitectura, un componente se construye para manejar aspectos de desarrollo específicos de una aplicación. MVC separa la lógica empresarial y la capa de presentación entre sí. Este patrón arquitectónico se utiliza principalmente para interfaces gráficas de usuario (GUI) de escritorio.

¿Qué es MVVM?

La arquitectura MVVM facilita una separación del desarrollo de la interfaz gráfica de usuario con la ayuda del lenguaje de marcado o código GUI. La forma completa de MVVM es Model–View–ViewModel.
El modelo de vista de MVVM es un convertidor de valor, lo que significa que es responsabilidad del modelo de vista exponer los objetos de datos del modelo de tal manera que los objetos se administren y presenten fácilmente.

Diferencias clave entre MVC y MVVM

  • El marco MVC es un patrón arquitectónico que separa una aplicación en tres componentes lógicos principales: Modelo, Vista y Controlador. Por otro lado, MVVM facilita una separación del desarrollo de la interfaz gráfica de usuario con la ayuda del lenguaje de marcado o código GUI.

  • En MVC, el controlador es el punto de entrada a la aplicación, mientras que en MVVM, la vista es el punto de entrada a la aplicación.

  • El componente MVC Model se puede probar por separado del usuario, mientras que MVVM es fácil para pruebas unitarias separadas y el código está controlado por eventos.

  • La arquitectura MVC tiene relaciones de "uno a muchos" entre Controller y View, mientras que en la arquitectura MVVC, "uno a muchos" relaciones entre View & View Model.
