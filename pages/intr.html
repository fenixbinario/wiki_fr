FENIX RELOAD

Introducción

¿Como funciona?

Fénix Reload se incrusta en el HTML formando parte del propio lenguage de marcado, hidratando el comportamiento de html5

¿Como se incrusta?

Se añade como cualquier otro ELEMENTO de HTML

Además tambien se añade su ATRIBUTO como cualquier otro ELEMENTO de HTML

Elementos

Se añade como cualquier otro elemento de HTML

  • <server></server> --> ELEMENTO --> En su interior se declaran los servidores a los que se conectará nuestra app/web. 

    
        <body>
            <server>
                <api host="domain.com" headers="content-type" role="admin"></api>
            </server>
        </body>

  • host="" --> ATRIBUTO --> En su interior se declaran la dirección del servidor. 

    
            <body>
                <server>
                    <api host="localhost:8080" ></api>
                </server>
            </body>

  • role="" --> ATRIBUTO --> En su interior se declara el nivel de privilegio de acceso al servidor. 

    
            <body>
                <server>
                    <api role="user" ></api>
                </server>
            </body>

  • auth="" --> ATRIBUTO --> es lo mismo que headers="" con la única diferencia que es para servidores websockets. 

    
            <body>
                <server>
                    <api host="ws://mydomain.co" auth="access_token" access_token="eyJh.2gY8Lp" role="user"></api>
                </server>
            </body>

  • layer="" --> ATRIBUTO --> sirve para declarar las capas que están dentro de un contenedor, estas capas pasan a convertirse en un elemento dentro del DOM. Pueden anidarse tantas veces como se queira. 

    
            <body layer="setting contact">
                <server>
                    <api host="http://" ></api>
                </server>
                <div>
                    <button type="button" view="body/setting" > SETTING <button>
                    <button type="button" view="body/contact" > CONTACT <button>
                </div>
                <setting>
                    <div></div>
                </setting>
                <contact>
                    <div></div>
                </contact>
            </body>

  • view="" --> ATRIBUTO --> Muestra la capa a la que nos queremos dirigir y la hace visible. . 

    
            <button type="button" view="body/contact" > CONTACT <button>

  • <role></role> --> ELEMENTO --> Muestra dentro de una capa que se muestra segun los previlegios diseñados para nuestra aap/web. 

    
            <body layer="setting">
                <setting>
                    <role role="" server=""></role>
                </setting>
            </body>

  • role="" --> ATRIBUTO --> Define el valor de privilegio del elemento <role> . 

    
            <body layer="setting">
                <server>
                    <api host="http://" ></api>
                </server>
                <setting>
                    <role role="invitado" server="api"></role>
                    <role role="admin" server="api"></role>
                    <role role="root" server="api"></role>
                </setting>
            </body>

  • server="" --> ATRIBUTO --> Define a que servidor se obtiene ese nivel de privilegio en el elemento <role>. 

    
            <body layer="setting">
                <server>
                    <api host="http://" ></api>
                    <websocket host="ws://" ></websocket>
                    <api2 host="http://" ></api2>
                </server>
                <setting>
                    <role role="admin" server="api"></role>
                    <role role="admin" server="websocket"></role>
                    <role role="admin" server="api2"></role>
                </setting>
            </body>

  • onload="" --> ATRIBUTO --> Carga un archivo .html dentro de los elementos <role>, <div>. 

    
            <body layer="setting">
                <server>
                    <api host="http://" ></api>
                </server>
                <setting>
                    <role role="invitado" onload="invitado.html" server="api"></role>
                    <role role="admin" onload="admin.html" server="api"></role>
                    <role role="root" onload="root.html" server="api"></role>
                </setting>
            </body>

  • lang="" --> ATRIBUTO --> Los elementos que contengan este atributo mostraran texto, imagenes y enlaces segun el idioma del navegador. si el elemento es un <role>, <div> se conporta como un onload="" con la diferencia que cargará un html u otro segun el idioma del navegador. 

    
            <p lang="hello" ><p>
            <img lang="hello" >
            <a lang="href" href="hello" >Link<a>
            <role role="invitado" lang="hello" server="api"></role>

  • data="" --> ATRIBUTO --> Define los datos que se incluiran en el cuerpo de un mensaje method, websocket. 

    
                <form data="role user pass">
                    <input role="" ></input>
                    <input user="" ></input>
                    <input pass="" ></input>
                </form>

  • method="" --> ATRIBUTO --> se utiliza para peticiones a una API, WEBSOCKET, MQTT, obtiene los datos que previamente se han declarado en el atributo body="". 

    
                method="get"
                method="onget"
                method="post"
                method="onpost"
                method="patch"
                method="put"
                method="delete"
                method="emit"
                method="on" 
                method="pub"
                method="sub"

    Para enviar datos a una API se puede acompañar al atributo method="" con los siguientes atributos

    
            host="domain" path="/user" print="form" println="step" body="div" clear="div"

    Para enviar datos a un server WEBSOCKET se puede acompañar al atributo method="" con los siguientes atributos

    
            host="domain" event="name-event"  print="form" println="step" body="div" clear="div"
